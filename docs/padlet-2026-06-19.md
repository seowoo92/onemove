** 개발내용 (6/19)

17. 화면별 디자인 시안 전면 적용
1) 웰컴(S0): 오르막 모티프 — 클레이 3점 브랜드 마크, 헤드라인, 오르막 점 패스(SVG)
2) 코치 선택(S1): 큰 아바타 + 하단 썸네일 전환, 시작 버튼 한글 조사 자동(로/으로)
3) 마음 날씨(S2): 날씨 카드 3종 + 안내 말풍선
4) 홈(S3): 딥그린 진행 카드, 완료 버튼 노랑(#F7EBBE), 완료 행, 영역칩 색, 쉬운 버전 카드
5) 코치 모달(S4): 말풍선 + 큰 상반신 코치 캐릭터(하단 경계 아래로 자연스럽게 블리드)
6) 완료 축하 카드: 풍경 이미지 배너 + 코치 안아주기 문구
7) 기록 탭: 범례·타임라인에 날씨 아이콘 적용
8) 설정 탭: 프로필 헤더 + 다시 고르기 박스 통일 + 위기상담 2칸 (계정 섹션은 프로필 카카오 버튼에 통합)

18. 용어 / 브랜딩 정비
1) "오늘 상태" → "마음 날씨"로 통일 (맑음 / 구름 / 비)
2) 코치 이름: 유쾌한 햇살 / 진중한 숲 / 다정한 물

19. 공통 UX 보강
1) 하단 탭바에 라인 아이콘 추가 (홈 / 기록 / 설정)
2) 스크롤 안내 화살표 (스크롤이 필요한 화면 하단, 끝까지 내리면 자동 숨김)
3) 데스크톱 좌측 소개 패널 리디자인 + QR 코드 (휴대폰으로 스캔해 모바일에서 바로 사용)
4) 모든 화면 상단 흰 띠 제거 (컨테이너 배경 크림으로 통일)

20. 이미지 에셋 처리
1) 마음 날씨 아이콘 3종 (배경 제거 + 256px)
2) 코치 캐릭터 PNG 3종 (상반신, 코치 모달에 활용)
3) 완료 축하 배너 (풍경 일러스트, 1.8MB → 820px·389KB 최적화)
4) QR 코드 생성 (라이브 주소)
- jimp / qrcode 임시 도구로 배경 제거·리사이즈·QR 생성

21. 버그 수정
1) 날짜 키 UTC → 로컬(KST) 기준으로 통일 (하루 경계·기록 키 어긋남 해결)
2) 기록 탭 완료수 누적("16/4") → 현재 완료 집합 덮어쓰기 (완료수 ≤ 총개수 보장)

** 트러블 슈팅 3

13. 한 화면을 채우는 화면이 위로 쏠리고 아래 여백 발생
문제: 웰컴·코치 선택처럼 한 화면을 꽉 채워야 하는 화면이 위로 몰리고 아래 큰 여백이 생김.
원인: 모바일/태블릿 컨테이너에 높이가 지정되지 않아 자식의 height:100%가 콘텐츠 높이로 무너짐.
해결: 컨테이너에 height:100%(부모 100vh 기준) + box-sizing:border-box 지정.

14. 모든 화면 최상단에 얇은 흰 띠
문제: 화면 맨 위에 흰색 띠가 보임.
원인: 컨테이너 상단 패딩 영역에 흰 바깥 배경(#FFFFFF)이 비침.
해결: 모바일/태블릿 컨테이너 + 데스크톱 베젤 안쪽 배경을 크림(#FAF6F0)으로 지정.

15. 날짜가 UTC 기준이라 화면 표시와 어긋남
문제: localStorage 날짜 키가 화면 표시 날짜와 달라, 자정~오전 9시(KST)에 하루 리셋이 어긋나거나 기록이 엉뚱한 날짜에 묶일 수 있음.
원인: 날짜 함수가 toISOString()(UTC)을 사용. 화면 표시는 로컬(KST)이라 최대 9시간 차이.
해결: 날짜를 로컬 기준 YYYY-MM-DD로 통일. 기록 저장 키도 동일하게 맞춤.

16. 기록 탭 완료수가 총개수보다 큼("16/4")
문제: 마음 날씨를 다시 고른 뒤 기록의 완료수가 총개수를 초과해 표시됨.
원인: 날짜별 기록이 완료 루틴 이름을 계속 누적하는데, 재선택 시 새 완료분이 같은 날짜에 쌓이고 총개수는 작은 값으로 덮임.
해결: 기록을 '현재 완료 집합으로 덮어쓰기'로 변경(누적 X → 완료수 ≤ 총개수). 재선택 시 당일 기록 초기화.

17. ChatGPT 생성 이미지가 투명 대신 체크무늬가 박혀 저장됨
문제: 마음 날씨 아이콘 PNG가 투명 배경이 아니라, 투명 표시용 체크무늬가 그림에 픽셀로 박혀 있음.
원인: ChatGPT 이미지 생성이 "투명 배경"을 실제 알파가 아닌 체크무늬 패턴으로 그려서 출력(같은 파일을 다시 받아도 동일).
해결: jimp로 테두리에서 flood-fill하여 배경 체크무늬만 투명 처리 + 256px 리사이즈.

18. 코치 상반신 캐릭터가 팝업에서 어색하게 잘리고 옆 텍스트 영역이 좁음
문제: 코치 모달에 큰 상반신 캐릭터를 넣으니 하단 여백·잘린 느낌이 나고, 오른쪽 텍스트가 두 줄로 줄바꿈됨.
원인: 상반신 PNG 하단 여백 노출 + PNG 좌우 투명 여백(약 20~24%) 미고려.
해결: overflow:hidden으로 캐릭터를 시트 하단 경계 아래로 블리드(자연스럽게 올라온 느낌), PNG 실제 콘텐츠 폭을 측정해 오른쪽 텍스트 영역을 투명 여백 위로 당겨 한 줄 유지.
