# 오늘만큼 — 발표 슬라이드 기획안 (수정본, 2026-06-19)

> 발표 시간: 총 10분 = 발표 ~7분 + Q&A 3분.
> 슬라이드 6이 "3화면×3세트 = 사실상 3장"이라 전체 ~12장 → 7분엔 빠듯. **슬라이드 6을 2장으로 압축**하거나 캡처 대신 **30~45초 라이브 데모**(마음 날씨→루틴→완료→코치 팝업)로 대체 권장.

## 1. 타이틀
**오늘 할 수 있는 만큼만, 오늘만큼** — 회복탄력성 AI 루틴 코치 서비스
- 말할 내용: 본인이 쉬었던 청년이었던 경험에서 출발. 무기력한 날에도 부담 없이 작은 루틴을 시작하도록 돕는 웹앱.

## 2. 문제 인식 / 해결 아이디어 (좌우 2단)
**문제**: 무기력한 날에는 '시작' 자체가 어렵다
- 무엇을 할지 정하는 것부터 부담 / 실패하면 죄책감 → 이탈 / 기존 앱은 계획·체크 중심이라 힘든 날의 회복 경험 부족

**해결**: 오늘 상태에 맞춰, 할 수 있는 만큼만 제안
- **마음 날씨**(맑음/구름/비)로 오늘 상태 체크 ← *"상태 체크"보다 따뜻한 메타포, 차별점으로 강조*
- 상태에 따라 루틴 개수·난이도 조절 / 어려우면 쉬운 버전으로 재도전 / AI 코치가 상황별 격려

## 3. 아이디어 선정부터 개발까지 (날짜별 타임라인)
> ⚠️ 발표가 6/22→**6/19로 당겨짐**. 디자인·버그·배포가 전부 6/19에 이뤄졌으니 타임라인을 6/19로 맞출 것.
- 6/12~14: PRD 작성, 루틴 28개 정의, 쉬운 버전 1:1 매칭
- 6/15~17: P0 MVP (상태 체크·루틴 추천·완료·AI 메시지)
- 6/18: 카카오 로그인, 기록 화면(recharts), 닉네임
- **6/19: 디자인 시안 전면 적용 + 이미지 에셋 + 버그 수정(날짜·완료수) + 반응형 마감 + 배포 + 발표**

## 4. 웹/앱 반응형 (좌우 2단)
- PC: 폰 베젤 + **좌측 소개 패널(브랜드 마크·슬로건·QR 코드 — 휴대폰으로 스캔해 바로 사용)** / 모바일: 전체 화면
- *"모바일 우선 웹앱인데 PC 방문자도 QR로 폰에서 바로 쓰게 한 배려" = 완성도 포인트*

## 5. 사용자 플로우 차트
- **v2.0 사용 (v1.0에서 갱신됨)**: S0 진입 추가 · 마음 날씨 · 카카오 로그인 완료 · 완료 축하 카드 · 하단 탭 반영

## 6. 화면 캡처 (하단에 기능/기술 1줄) — *2장으로 압축 권장*
1) 온보딩(진입) / 코치 선택 / 마음 날씨
2) 오늘의 루틴(홈) / 쉬운 버전 전환 / **AI 코치 팝업(말풍선+캐릭터, 최신 디자인)**
3) **완료 축하 카드(풍경 배너)** / 기록 / 설정
- *완료 축하 카드는 감정적 마무리 비주얼로 강력 — 꼭 포함*

## 7. 핵심 기술 구조 및 구현 시 고려한 점
- Vite + React / Tailwind CSS / localStorage / Solar API / Supabase Auth / GitHub Pages
- 회원가입·서버 DB 제외, localStorage 우선 / 푸시 알림 대신 카카오 리마인드(예정) / streak 제외(죄책감) / 이유 선택 생략, 쉬운 버전으로 단순화
- **루틴 추천 = 규칙 기반**(개수·난이도·영역 분산·어제 제외), AI는 코치 메시지에만 사용 ← *Q&A에서 "추천도 AI냐" 물으면 명확히 답할 것*
- 말할 내용: 복잡한 서버보다 MVP 완성을 우선, 기록은 localStorage. AI 메시지는 Solar, 실패 시 예비 메시지로 전환.

## 8. AI 활용 포인트
- 코치 성격 반영: 유쾌한 햇살 / 진중한 숲 / 다정한 물 ← *풀네임으로*
- 상황 반영: 완료 / 쉬운 버전 완료 / 쉬어가기 / 전체 완료
- 루틴명 반영: 수행 행동에 맞춘 메시지 / 예비 메시지 폴백으로 시연 안정성

## 9. 앞으로 디벨롭할 부분 (8월~)
