# 개발일지 2026-07-23 (목) — 루틴 카드 대개편 + 자율성 기능 3종 (새벽 세션, 커밋 8개)

## 1. 개발 환경 정비 (점검 중 발견)
- **.env.local의 VITE_SUPABASE_ANON_KEY가 비어 있던 문제 복구** (7/14 서버 이관 때부터) — 로컬 dev에서 Solar AI가 전부 예비 메시지로만 나오고 로그인·동기화가 죽어 있었음. 배포 번들에서 공개 anon 키를 추출해 채움. 배포 사이트는 원래 정상
- index.html 파비콘·manifest **상대경로화** — dev 서버에서 base 이중 결합(`/onemove/onemove/`)으로 404 나던 콘솔 에러 해결, 프로덕션 동작 동일

## 2. 시연 동선 사전 점검 (웹, puppeteer 통짜 리허설)
- 온보딩→마음날씨→완료/어려워요/쉬어가기→완료축하+하루마무리→정원 맞이하기→기록(실·데모)→앨범 전 구간 자동 리허설 — **이슈 0건 통과**
- `?record=demo`가 정원에 새지 않는 것 확인. 카톡 수신 단계만 실기기 필요(미검증)

## 3. 쉬운버전 표시 정직화 + 원래 난이도 왕복
- 문제 인식(사용자 실기기 발견): 시스템이 자동으로 쉬운 버전을 뽑아도 "더 쉬운 버전으로 **바꿨어요**"로 표시 — 안 한 행동을 했다고 말하는 워딩
- 자동/사용자 전환을 `onemove_easy_auto`로 구분. **최종 확정: 자동 카드는 배지 없음**(일반 카드와 동일 밀도), 사용자가 '지금은 어려워요'로 바꾼 카드에만 "바꿨어요" 배지(행동 피드백)
- **"원래 난이도로 ⇄"** 링크(칩 행, 위아래 맞바꿈 화살표) — 쉬운버전↔보통 왕복 가능
- 서버 동기화: `daily_entries.easy_auto_ids` 열 추가(사용자가 SQL 실행) — 기기 간 표시 일치

## 4. '다른 루틴으로 바꾸기' — 하기 싫은 루틴 대응 (논의 후 신규 기능)
- 논의 결론: '할 수 없는'(어려워요)과 별개로 '하기 싫은' 루틴엔 **바꿀 수 있되 무한하지 않게**
- 활성 카드 **새로고침 아이콘** → 같은 난이도의 다른 루틴으로 교체. **카드당 1회 제한**(회피·무한리롤 방지), ★ 매일 루틴 제외, 오늘 세트·내보낸 루틴 재등장 방지, 영역 겹침·어제 루틴 단계적 회피 (`pickSwapCandidate`)
- 동기화: `daily_entries.swap_used_ids`·`swapped_out_ids` 열 추가(사용자가 SQL 실행)

## 5. 영역 칩 표시명 (사용자 확정)
- 단독 노출 시 뜻이 통하게 화면 라벨만 보완: 몸 깨우기→**몸 움직이기**, 바깥→**바깥 활동**, 공간→**공간 정리**, 연결→**마음 연결** / 에너지·자기돌봄·성취는 그대로
- 교훈: 선택지 "에너지 유지"(=그대로 두기)를 새 표시명으로 오독해 반영했다가 정정 — 옵션 라벨은 중의성 없게

## 6. 루틴 카드 컴팩트 대개편 (시안 5종 비교 후 확정)
- 구조: **칩 행(좌 ★·새로고침 / 우 난이도·영역 칩) → 루틴명 한 줄(말줄임) → 버튼 행(우측 정렬)**
- 버튼: **완료=노랑 필**(부드러운 톤 유지, 사용자 확정) / **보조=회색 테두리 아웃라인 필** — 주/보조 위계 분리, 슬림(7px/6px 패딩)
- **"좋아요" 4개가 실기기 실효 높이(760px 검증)에서 진행 카드 포함 스크롤 없이 한 화면**
- **완료·쉬어가기 행 상단 정렬** + **왼쪽 체크 원 탭 → 체크 해제·카드 복원**(완료 실수 방지, X 아이콘은 삭제로 오인되어 제거, 기록·정원 카운트 동시 복원) — 사용자 안내서 기재 필요
- 오늘 탭 헤더를 공통 ScreenHeader(23px)로 통일 — 4개 탭 타이포·위치 동일

## 7. 카톡 루틴 카드 수동 재발송
- 홈 하단 "**오늘 루틴 카톡으로 받기**"(알림 동의 사용자만, 2단 가운데 정렬) — 교체·난이도 조절로 구성이 바뀐 뒤 현재 구성 그대로 재발송(`resendRoutineCard`, 하루 1회 제한 없음·사용자 탭만). **실기기 수신 확인 필요**
