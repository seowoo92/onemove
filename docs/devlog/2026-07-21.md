# 개발일지 2026-07-21 (화) — 맞이하기 인터랙션 + 요소 순서 개편 + CBT 명문화 + 외부 디자인 피드백 반영

## 1. 정원 '맞이하기' 인터랙션 (사용자 아이디어)
- 새 요소를 자동 배치하지 않고 **"정원을 채워나갈 요소가 생겼어요" 필이 깜빡임 → 화면 탭 시 순차 등장**
- 등장 연출 톤 교정: 바운스 팝업 → 1.1초 페이드+떠오름, 스태거 0.22초 (차분하게)
- 맞이하기 전 이탈 시 다음 방문에도 대기 유지. ?garden 데모에서도 동일 연출 (발표 킬러 장면: 탭 → 정원 만개)

## 2. 요소 노출 순서 개편 — 작은 것부터
- 새싹(2)→나비(5)→수국(9)→새(14)→햇님(20)→구름(27)→해바라기(36)→작은나무(47)→연못(60)→큰나무(75)
- 임계값 근거 재확인: 계절 92일·총 75개 = 하루 0.8개(느슨한 완주 설계), 초반 촘촘·후반 느슨(간격 3→15)
- 등장 문구 확정: 구름 "두둥실 나타나요", 큰 나무 "생겨나요" 등 요소별 고정

## 3. CBT 정합성 명문화
- 현재 기능 전수 점검: 행동 활성화·과제 단계화·기분 모니터링·재해석·강화 모두 부합, 어긋남 없음
- CLAUDE.md 주의사항에 **새 기능 CBT 체크 원칙** 추가 (페널티·실패 강조·상향 압박·비교 랭킹·"치료" 표현 금지)
- 발표 표현 가이드: "치료"가 아니라 "CBT의 행동 활성화 원리를 적용" + 위기 연락처 = 윤리성 어필

## 4. 외부 디자인 피드백 5건 반영 (검토→미리보기→컨펌→적용 프로세스)
- 크기 위계: 큰나무 -10%·작은나무 -15% (비율 역설적으로 개선), 해바라기 상향
- 깊이감: 접지 그림자를 수관 폭(76%)→**밑동 폭(56%) 기준**으로, blur 증가 — 스티커 느낌 해소
- 하늘 분산: 구름 하강, **나비 -15도 회전**(rotate 필드 신설)·중앙 이탈
- 연못 축소·좌측 이동, 수국 상향(연못 위로 드러남), 우하단 밀집 해소
- 구름 지적은 정지컷 전제였음을 식별 (실제는 75초 횡단) — 피드백 수용 시 구현 특성 대조 필요
