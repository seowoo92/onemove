## 공통 레이아웃 / UX
- **AppLayout 모바일·태블릿 컨테이너 `height:100%` 보정**: 한 화면을 채우는 화면(웰컴·코치선택 등)이 위로 쏠리고 아래 여백이 생기던 문제 해결 (부모 100vh 기준 + box-sizing border-box)
- **모든 화면 상단 흰 띠 제거**: 모바일/태블릿 컨테이너 + 데스크톱 베젤 안쪽 배경을 크림(#FAF6F0)으로 통일 (paddingTop 영역에 흰 바깥 배경이 비치던 문제)
- **하단 탭바 라인 아이콘 추가** (TabBar.jsx): 홈(집)/기록(문서)/설정(슬라이더) SVG 라인 아이콘 + 라벨, 활성 #24523F·비활성 #9AA39C
- **스크롤 안내(ScrollHint, AppLayout 내부)**: 아래로 더 스크롤할 내용이 있을 때 하단 중앙에 딥그린 ⌄ 버블 표시, 끝까지 내리면 자동 숨김. 모바일(window 스크롤)·베젤(내부 스크롤) 양쪽 대응, 콘텐츠 늦은 렌더 대비 주기 재측정(400ms)
- **데스크톱 좌측 소개 패널 리디자인** (tandanji 레퍼런스 참고): 클레이 3점 브랜드 마크 + 헤드라인 + 핵심 3가지 + 구분선 + **라이브 QR 코드**(qr-onemove.svg, 휴대폰 스캔 유도) + 슬로건("오늘만큼, 딱 그만큼 / One move a day")

## 화면별 디자인 적용
- **웰컴 S0 (WelcomeScreen)**: 불릿 안내 카드 → **오르막 모티프**(클레이 3점 마크, 헤드라인 "오늘 할 수 있는 / 만큼만.", 서브카피, 오르막 점 패스 SVG, 카카오/로그인없이 버튼). 이후 미세조정: 헤드라인 2줄(데스크톱 패널과 통일), 서브문구 줄바꿈 정리, 점 패스 확대, 카카오 버튼 축소
- **코치 선택 S1 (CoachSelect)**: 큰 아바타(200px) + 썸네일(74px), 제목 "함께할 코치를 골라요", 시작 버튼 한글 조사 자동(로/으로 — 받침 판별), 전체 세로 중앙 정렬
- **마음 날씨 S2 (StateCheck)**: 날씨 카드 3종 + 안내 말풍선. 아이콘은 임시 CSS 아트 → **실제 클레이 PNG**(weather-sunny/cloudy/rainy)로 교체, 연한 색 동그라미 위에 배치
- **홈 S3 (Home)**: 인사 헤더 + 딥그린 진행 카드("한 걸음씩"·그라데이션 바) + **완료 버튼 노랑(#F7EBBE)** + 완료 행(연녹·체크원·취소선) + 영역칩 카테고리 색(몸돌봄 초록/환경 파랑/마음 코랄) + 쉬운버전 카드(별 배지)
- **코치 메시지 모달 S4 (CoachModal)**: 바텀시트 다듬기 → **스케치 디자인**(메시지 말풍선+꼬리 + 큰 코치 상반신 캐릭터가 하단 경계 아래로 블리드해 올라옴 + 이름·계속하기). 캐릭터 1.5배 확대, PNG 투명 여백(약 20%) 측정해 오른쪽 텍스트 영역 폭 확보
- **완료 축하 카드 (Home 하단, allResolved 시)**: 해질녘 밴드(시안 B) → **컴팩트 + 풍경 이미지 배너**(complete-banner.png 전체 표시, 잘림 없음) + "오늘 하루도 잘 해냈어요" + 코치 안아주기 문구(조사 자동)
- **기록 탭 (RecordScreen)**: 범례·타임라인 좌측 레일을 색 점 → **날씨 아이콘**으로 교체
- **설정 (SettingsScreen)**: 시안 재구성 — 상단 **프로필 헤더 카드**(닉네임+변경+"○○ 코치와 함께"+카카오 버튼), 알림 토글+안내 문구, **AI 코치 다시 고르기**·**마음 날씨 다시 고르기** 동일 디자인 흰 박스(코치 선택 화면 이동), **위기상담 연한 회색 2칸 카드**(전화 아이콘·tel:), **계정 섹션 삭제**(로그인/로그아웃은 프로필 카카오 버튼에 통합), 전체 컴팩트화로 **스크롤 없이 한 화면**

## 버그 수정
- **날짜 버그**: storage.js `today()`가 `toISOString()`(UTC)을 써서 화면 표시(로컬 KST)와 어긋남 → **로컬 기준 YYYY-MM-DD로 통일**. 자정~오전9시 KST 사이 하루 경계·기록 키 어긋남 해결. Home의 history dateKey도 storage.getTodayKey() 사용
- **"16/4" 완료수 누적**: addHistoryEntry가 이름을 계속 누적 → 마음 날씨 재선택 시 같은 날짜에 쌓여 완료수 > 총개수. **setHistoryEntry로 현재 완료 집합 덮어쓰기**(완료수 ≤ 총개수 보장) + 재선택 시 removeHistoryEntry로 당일 기록 초기화
