## 재정비 기간 활동 (6/20 ~ 7/12)

### 7/6 — 서비스 장애(흰 화면) 진단·복구
- 증상: 배포 페이지 접속 시 흰 화면. 프론트엔드(GitHub Pages)는 정상(HTTP 200, 콘솔 에러 0)인데 화면이 뜨지 않음
- 원인: **Supabase 무료 플랜이 7일 미사용으로 프로젝트를 자동 일시정지(pause)** → 저장된 카카오 로그인 세션이 있는 브라우저에서 `supabase.auth.getSession()`의 토큰 갱신 요청이 응답 없이 대기 → `setScreen()`이 실행되지 못해 화면 결정이 영영 안 됨. 게스트(세션 없음)는 정상이라 원인 파악이 어려웠음
- 조치: Supabase 대시보드에서 프로젝트 Resume → 즉시 복구 확인
- 교훈: ① 흰 화면 신고 시 코드보다 **Supabase 정지 여부를 먼저 확인** ② 백엔드 장애가 앱 전체 사망으로 이어지지 않도록 초기화 안전장치 필요 → 오늘 코드 하드닝으로 반영

## 2차 스프린트 확정 스코프 (7/13 ~ 7/28 예선 마감)
- **localStorage → Supabase 전환** (로그인 사용자 데이터 동기화) — 수업에서 배운 도구 활용(평가기준 1번)
- **카카오 '나에게 보내기'(talk_message) 알림**: ① 루틴 확정 시 오늘의 루틴 카드 발송 ② 오후 미완료 루틴 리마인더(Edge Function 스케줄)
- 디자인 리뉴얼 + 탭 4개 개편(오늘·기록·코치·설정) + 기록 탭 대시보드 승격(캘린더형 마음 날씨)
- Web Push는 카카오 알림으로 대체(컷), PWA는 manifest(홈 화면 설치)만
- 여유 시: 완료 무지개 효과, 랜덤 힐링 문구, 루틴 확장
