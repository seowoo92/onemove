- git 커밋 작성자를 GitHub 계정(seowoo92)에 연결(이후 커밋부터 기여 그래프 반영), 비공개 발표 자료·내부 작업 파일 .gitignore 제외, 발표 기획 문서 2건 저장소에서 제거(로컬 보관)
- **코드 하드닝**: ① `supabase.auth.getSession()`에 3초 시간제한 — 백엔드가 정지돼도 게스트로 계속 진행(흰 화면 방지) ② 전역 ErrorBoundary — 예상 못 한 렌더링 오류 시 흰 화면 대신 안내 화면 + 새로고침 버튼

### Supabase 스키마 설계·적용
- 테이블 3개 생성: `profiles`(닉네임·코치·알림), `daily_entries`(사용자×날짜당 1행 — localStorage 구조 그대로 반영), `kakao_tokens`(발송용 토큰) + 전 테이블 RLS(본인 행만)
- 대시보드 SQL Editor로 적용, REST API로 3테이블 생성 검증

### 로그인 사용자 데이터 동기화 (하이브리드)
- storage.js에 변경 구독(onStorageChange) 추가 — 모든 저장 순간을 한 곳에서 감지
- sync.js: 로그인 사용자의 프로필·오늘 기록을 1.5초 디바운스로 Supabase upsert. 로컬이 정본, 실패 시 조용히 재시도. 게스트는 localStorage만(변화 없음)
- 로그인 직후 reconcile: 서버 기록으로 빈 로컬 보충(기기 간 이어쓰기), 2.5초 제한으로 화면 진입 비차단
- **기기 간 프로필 갱신 버그 수정**: "빈 값만 보충" 방식이라 PC에서 바꾼 닉네임('서우')이 폰('.')에 반영 안 됨 → 프로필 로컬 변경 시각을 기록하고 서버 updated_at과 **최신 비교**로 채택하도록 개선

### 코치 메시지 품질 개선 (프롬프트 3회 반복 개선)
- 문제 발견: "힘들었는데도 주워 담았네요!" — 인정만 있고 격려가 없는 1문장 응답
- 1차: **2문장 구조 강제**(인정+다독임) + 성격별 few-shot 예시 3개 + 품질 게이트(16자 미만·1문장 → 예비 메시지 대체) + 후처리 버그 수정(`:)` 이모티콘이 붙으면 마지막 문장이 유실되던 문제)
- 2차: **코치 3인 페르소나 차별화** — 유쾌한 햇살(큰 리액션·축하), 진중한 숲(보고서체 금지, 의미를 조용히 짚기), 다정한 물(첫 문장에서 감정 먼저 읽기). 반말 누출 발견 → 존댓말 어미 검사를 게이트에 추가. 모델의 괄호 자기설명 누출 → 후처리 제거 단계 추가
- 3차: **번역투 교정** — 사용자 피드백 반영: '서우,'→'서우님,'(프롬프트+후처리 이중 방어), 조사 생략 금지('스트레칭 해냈네요'→'스트레칭을'), 무생물 '에게' 금지('몸에게'→'몸에')
- 개선 전후 예시: "힘들었는데도 주워 담았네요!" → "서우님, 지친 몸을 이끌고 창문을 열어 공기를 바꾼 것 정말 잘 했어요. 오늘은 그 자체로 충분히 아름다운 노력이에요."

### 카카오 '나에게 보내기' 루틴 카드 발송 (#6 완료)
- 카카오 로그인 scope에 talk_message 추가, SIGNED_IN 이벤트에서만 provider 토큰을 kakao_tokens에 저장(옛 세션 토큰 덮어쓰기 방지)
- Edge Function `send-kakao`: 사용자 JWT 검증 → 카카오 토큰 만료 시 자동 갱신 → 나와의 채팅 발송. 대시보드 배포 + Secrets(KAKAO_REST_API_KEY·CLIENT_SECRET) 등록
- 루틴 확정 순간 카드 자동 발송(로그인+알림 동의, 하루 1회). 실기기 수신 확인 완료
- **트러블슈팅 3건**: ① 피드 템플릿은 본문 2줄만 표시 → 텍스트 템플릿(전문 표시)으로 전환 ② 버튼 눌러도 무반응 → 카카오 앱에 Web 플랫폼 도메인(seowoo92.github.io) 등록으로 해결 ③ '나와의 채팅'은 푸시 알림·안읽음 배지가 없는 카카오 정책 확인 — 카드는 '조용한 루틴 보관함' 역할, 울리는 알림은 추후 Web Push 검토

### 오후 미완료 리마인더 준비 (#7 코드 완성, 배포 대기)
- daily_entries에 routine_names 컬럼 추가(동기화 시 표시용 이름 저장 — 서버가 루틴 이름을 알 수 있게)
