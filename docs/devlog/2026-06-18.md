### 트러블슈팅 — KOE205 에러
- 증상: 카카오 로그인 시 "설정하지 않은 동의 항목: account_email" 에러
- 원인: Supabase Auth가 account_email 스코프를 서버에서 강제 요청하나, 카카오는 개인 개발자 앱에 이메일 항목을 허용하지 않음. 프론트에서 scope를 profile_nickname만 요청해도 Supabase 서버가 강제 추가하여 코드로는 해결 불가 (Supabase 공식 이슈 #36878)
- 해결: 카카오 비즈앱 전환 + 이메일 동의항목 신청 → 정상 로그인 구현
  - 비즈앱 전환에 앱 아이콘이 필요하여 아이콘 제작과 연계
- account_email은 선택 동의로 유지 (필수 동의 미적용)

### 닉네임 설정 기능
- 설정 탭에서 앱 내 닉네임 직접 설정 (localStorage: onemove_nickname, 최대 12자)

- 계정 표시: 닉네임 있으면 "OO님으로 로그인됨", 없으면 "로그인됨"
- 저장 버튼 클릭 시 "저장되었어요" 2초 표시
- 코치 메시지에 닉네임 반영
  - generateCoachMessage에 nickname 파라미터 추가 및 전달 (Home.jsx → solar.js)
  - 시스템 프롬프트에 "자연스러울 때만 이름 사용, 매 문장 강제 삽입 금지" 규칙 추가
  - fallback 메시지: 닉네임 있으면 앞에 "${nickname}님, " 접두 후 cleanMessage 통과
  - 닉네임 없으면 기존처럼 이름 없이 생성

### 트러블슈팅 — 코치 메시지 닉네임 어색함
